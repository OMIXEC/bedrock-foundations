# Enterprise Fine-Tuning on Amazon Bedrock

### Production-Grade Custom Model Training Pipeline

## Overview

End-to-end, production-ready fine-tuning workflow for Bedrock custom models. Built for enterprise use with full observability, error handling, IAM least-privilege, and automated cleanup.

## Architecture

```
CSV Dataset → Quality Filter → Train/Val Split → JSONL → S3 (versioned)
→ Bedrock Fine-Tuning Job (with validation) → Custom Model
→ Provisioned Throughput → Inference
```

## Configuration via `.env`

Copy `.env.example` to `.env` and fill in your values:

```bash
AWS_REGION=us-east-1
FINETUNE_S3_BUCKET=bedrock-finetuning-enterprise-YOUR_SUFFIX
DATASET_PATH=dataset/data.csv
BASE_MODEL_ID=cohere.command-light-text-v14:7:4k
FINETUNE_JOB_NAME=enterprise-finetune-v1
CUSTOM_MODEL_NAME=enterprise-model-v1
IAM_ROLE_NAME=BedrockEnterpriseFinetuneRole
EPOCH_COUNT=5
BATCH_SIZE=8
LEARNING_RATE=0.00003
POLL_INTERVAL_SEC=60
POLL_TIMEOUT_SEC=7200
# After provisioned throughput is created:
CUSTOM_MODEL_ARN=arn:aws:bedrock:us-east-1:123456789:provisioned-model/xxxx
```

## Dataset Format

| Column     | Description                            |
| ---------- | -------------------------------------- |
| `dialogue` | Patient-doctor conversation text       |
| `note`     | Clinical note summarizing the dialogue |

Minimum quality thresholds applied automatically:

- `dialogue` length > 50 chars
- `note` length > 20 chars

## Key Enterprise Features vs. Foundations Template

| Feature         | Foundations | Enterprise                  |
| --------------- | ----------- | --------------------------- |
| Config source   | Hardcoded   | `.env` / environment vars   |
| Train/Val split | ❌          | ✅ 80/20                    |
| S3 versioning   | ❌          | ✅                          |
| IAM policy      | Broad       | Least-privilege             |
| Job polling     | Simple loop | Timeout-aware               |
| Error handling  | Basic       | Full `ClientError` catching |
| Inference cases | Single      | Multi test-case             |
| Cleanup         | Manual      | `CLEANUP_ENABLED` flag      |

## Hyperparameter Defaults

| Parameter      | Value | Notes                         |
| -------------- | ----- | ----------------------------- |
| `epochCount`   | 5     | Higher for enterprise quality |
| `batchSize`    | 8     | Conservative for stability    |
| `learningRate` | 3e-5  | Lower for larger datasets     |

## Prerequisites

- Amazon Bedrock access with Cohere `command-light-text-v14` enabled
- IAM permissions: `bedrock:*`, `s3:*`, `iam:CreateRole`, `iam:PutRolePolicy`
- Python 3.10+ with: `boto3`, `pandas`, `python-dotenv`
