# Bedrock Custom Model Fine-Tuning

A step-by-step guide for fine-tuning a foundation model on Amazon Bedrock using the Cohere `command-light-text-v14` model for clinical note generation.

## Use Case

**Medical Dialogue Summarization** — Fine-tune to generate structured clinical notes from patient-doctor conversations.

## Architecture

```
Dataset (CSV) → JSONL Formatting → S3 Upload → Bedrock Fine-Tuning Job → Custom Model → Provisioned Throughput → Inference
```

## Project Structure

```
bedrock-custom-finetuning/
├── fine_tuning_bedrock.ipynb   # Main notebook
├── dataset/
│   ├── data.csv                # Your training data (add here)
│   └── clinical_notes_fine_tune.jsonl  # Generated by notebook
├── images/                     # Screenshots for reference
└── README.md
```

## Quick Start

1. **Add training data** — Place your `data.csv` in the `dataset/` folder\
   - Required columns: `dialogue`, `note`
2. **Run the notebook** — Execute cells in order
3. **Monitor job** — Check status in AWS Console or Step 7 of the notebook
4. **Purchase Provisioned Throughput** — Required before running inference
5. **Run inference** — Paste your model ARN in Step 8

## Dataset Format

| Column     | Description                            |
| ---------- | -------------------------------------- |
| `dialogue` | Patient-doctor conversation text       |
| `note`     | Clinical note summarizing the dialogue |

## Hyperparameter Defaults

| Parameter      | Value | Notes                        |
| -------------- | ----- | ---------------------------- |
| `epochCount`   | 3     | Increase for larger datasets |
| `batchSize`    | 16    | Reduce if OOM                |
| `learningRate` | 5e-5  | Standard starting point      |

## Prerequisites

- Amazon Bedrock access with Cohere `command-light-text-v14` enabled
- IAM permissions: `bedrock:*`, `s3:*`, `iam:CreateRole`
- Python 3.8+ with `boto3`, `pandas`
